## Git Conventions

- **Commit messages**: Use conventional commits (`feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`)
- **Branch naming**: `{type}/{short-description}` (e.g., `feat/user-auth`, `fix/login-bug`)
- **PR size**: Keep PRs focused and reviewable (< 400 lines changed when possible)
